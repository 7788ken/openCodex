# T008 — Gateway Spike

## Objective

Explore a future gateway layer for openCodex without changing the MVP engine strategy.

## Scope

- Study always-on session routing patterns.
- Identify how external chat or web entry points could trigger openCodex tasks.
- Keep this work isolated from the first local CLI milestone.

## Acceptance Criteria

- The spike produces an architectural note, not production code.
- The result explains what belongs in Phase 2 instead of MVP.
- The proposal keeps Codex CLI as the local execution core.
