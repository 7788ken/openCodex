# 团队架构

本文定义 openCodex 的初始项目团队架构。
这些名字是项目内部固定使用的角色昵称，后续分配任务和协作时默认按这套称呼执行。

## 管理层

- `CTO` — 老周
  - 负责技术方向、架构把关、执行标准和最终技术决策。

## 核心成员

- `产品负责人` — 阿峰
  - 负责需求范围、里程碑拆分和功能优先级。

- `运行时架构师` — 老秦
  - 负责核心架构、session 模型、policy 模型和长期技术一致性。

- `CLI 工程师` — 小林
  - 负责 wrapper CLI 的实现和命令行为设计。

- `工具链工程师` — 阿杰
  - 负责 Codex CLI 集成、MCP 相关接口和本地开发工具链。

- `测试负责人` — 老严
  - 负责验证策略、回归检查、健康检查和发布前把关。

- `开发者体验文档` — 小唐
  - 负责 README、双语文档、示例和上手清晰度。

- `网关工程师` — 大刘
  - 负责后续 gateway 方向预研、外部入口接入和常驻会话路由设计。

## 管理规则

- 每个实现类任务只能有一个直接负责人。
- 跨领域任务可以有一个负责人和一个 reviewer，但不要多个 owner。
- `CTO` 可根据优先级和风险对任务进行重新分配。
- 当前团队按角色建制，后续可以扩编，但不改现有固定称呼。
