# CTO Soul Document

`prompts/cto-soul.md` is the editable initialization prompt for the openCodex CTO main thread.

## Purpose

- define the long-lived CTO identity
- keep the default template grounded in the general-purpose Codex CLI personal assistant persona
- pin the orchestration style for Telegram CTO workflows
- keep Codex CLI as the local execution engine while openCodex stays thin
- state that the CTO identity belongs to the host-level supervisor, not to sandbox child sessions
- keep chat, discussion, and research as first-class CTO behaviors before orchestration when appropriate
- shape how the CTO infers intent, delegates work, and decides when to ask the CEO

## Current Behavior

- Telegram CTO planning loads `prompts/cto-soul.md` before each planning turn
- the loaded document is appended to the CTO main-thread system prompt
- the host supervisor remains the CEO-facing CTO identity and workflow owner
- the CTO can stay in direct discussion/research mode instead of forcing every message into task orchestration
- sandbox child sessions are generated by the host supervisor as advisor, planner, reviewer, or helper sessions
- worker prompts are still generated by the CTO main thread, not by child agents
- if the file is missing, openCodex falls back to a built-in default soul template
- the tray app can open the file directly through `Edit CTO Soul`
- the tray app can also restore the default template through `Restore Default CTO Soul`

## Editing Rules

- keep the content in English
- keep the instructions stable and role-oriented
- prefer high-level policy over task-specific commands
- use this file to shape the host-level CTO supervisor, not to store temporary task notes
