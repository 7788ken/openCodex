# Project Overview

## Positioning

openCodex is a repo-aware coding agent project inspired by openclaw.
It is designed to help developers move from intent to validated code changes with a tight, local-first workflow.

## Design Goals

1. Keep the loop practical and observable.
2. Prefer minimal, reversible changes.
3. Combine planning, execution, and verification.
4. Stay friendly to local development workflows.

## Proposed MVP Capabilities

- Parse user tasks into structured steps.
- Search the repository for relevant files and symbols.
- Generate and apply minimal patches.
- Run focused commands for verification.
- Summarize what changed and what remains risky.

## Non-Goals for the First Version

- Large-scale autonomous refactors.
- Cloud-only execution.
- Full IDE replacement.
